## 1단계: 코드 분석

### 구조 파악
- **형식**: 단일 HTML 파일 (1048줄)
- **기술 스택**: HTML5 Canvas 2D API, Vanilla JavaScript
- **렌더링**: requestAnimationFrame 기반 게임 루프
- **상태 관리**: gameState 객체 중심
- **UI**: Modal 기반 메뉴 시스템 (Start, GameOver, Victory)

### 주요 컴포넌트
- Player: 20x20 사각형, 물리 기반 이동/점프
- Platforms: 6가지 타입 (NORMAL, MOVING, SPRING, DISAPPEARING, BREAKABLE, SLANTED)
- Enemies: 3가지 타입 (PATROL, CHASE, SHOOTER)
- Collectibles: 동전(50점), 보석(200점)
- Canvas: 800x600 고정 크기

## 구체적 개선 지시

### 우선순위 1: 핵심 기능 구현 (MUST)

#### 1-1. AI 기반 난이도 자동 조절 구현
**위치**: game loop 내 또는 completeStage() 후
**문제**: SPEC 기능 5 미충족. 현재는 "난이도 선택"일 뿐 "자동 조절"이 아님
**해결 방법**:
- `gameState.playerStats` 객체 추가: `{deathCount, timePerStage, comboMax, lastStageCleared}`
- `completeStage()` 후 성공률 계산 (예: 생명 손실이 없으면 100%, 1개 손실하면 66%)
- 성공률에 따라 다음 스테이지 적 개수 조정 (50% 미만이면 -1, 90% 이상이면 +1)
- 난이도 상향/유지/하향 메시지 표시 (현재 message 요소 활용)

#### 1-2. 스코어보드 구현 (localStorage 기반)
**위치**: 새 파일 또는 gameState 확장
**문제**: SPEC 기능 8 미충족. 플레이어 기록 없음
**해결 방법**:
- `localStorage.setItem('pixelQuestScores', JSON.stringify(scoresArray))` 구현
- `endGame()` 호출 시 현재 점수를 상위 10개와 비교하여 저장
- 게임 오버 모달에 "Your Rank: #3 / All Time" 추가 표시
- 메인 메뉴에 "HIGH SCORES" 버튼 추가

#### 1-3. SHOOTER 적 구현
**위치**: updateEnemies() 내 새로운 분기
**문제**: SPEC 기능 3 미충족. 3가지 타입 중 1개 부재
**해결 방법**:
- `enemy.type === enemyTypes.SHOOTER` 분기 추가
- PATROL처럼 경로 움직임 + 주기적 projectile 발사 (shootTimer 이용)
- Stage 4, 5에서 SHOOTER 타입 적 생성
- projectile 배열로 발사체 관리, player와 충돌 감지

#### 1-4. 생명 회복 아이템 구현
**위치**: collectibles 시스템 확장
**문제**: SPEC 기능 7 미충족. 생명 회복 메커닉 없음
**해결 방법**:
- collectible.type에 'star' 추가 (value: 'life')
- createCollectible()에서 type이 'star'인 경우 처리
- updateCollectibles()에서 star 수집 시 gameState.lives++ (최대 3)
- Stage 3, 4, 5에서 각 1개씩 star 배치 (확률 기반 또는 고정)

### 우선순위 2: 설계서 구현 완성도 개선

#### 2-1. 착지 충격 이펙트 추가
**위치**: updatePlayer() 플랫폼 충돌 감지 부분
**문제**: "착지 시 충격파 이펙트"가 없어 피드백 부족
**해결 방법**:
- 플랫폼 착지 시 `createImpactEffect()` 호출
- 원형 파동 2~3개를 ctx.arc()로 그려 확대 애니메이션
- 먼지 입자 5~10개를 소수점 x,y + 속도로 낙하
- 프레임 수로 애니메이션 진행 (총 10프레임)

#### 2-2. 벽 타기 메커닉 추가
**위치**: updatePlayer() 내 새로운 충돌 감지
**문제**: "수직 벽면에서 미끄러지며 점프 가능" 기능 미구현
**해결 방법**:
- 일부 플랫폼의 좌우 가장자리를 "벽"으로 정의 (예: 플랫폼의 x, x+width 경계)
- player.x가 벽에 닿고 player.velocityY > 0이면 velocityY를 0.3배로 감속
- 벽 타는 중 space 입력 시 점프 (velocityY = -12, x를 벽에서 멀어지게)
- 현재 단계: Stage 3~5에 벽 추가

#### 2-3. SLANTED 플랫폼 물리 구현
**위치**: updatePlayer() 플랫폼 충돌 부분
**문제**: 플랫폼 타입은 정의되었으나 로직이 없음
**해결 방법**:
- SLANTED 플랫폼의 기울기 angle 값 추가
- 플레이어가 SLANTED 위에 있을 때 velocityX += gravity * sin(angle) 적용
- 시각적으로 각도가 있는 사각형으로 그리기 (ctx.transform 이용)

### 우선순위 3: 독창성 및 디자인 강화

#### 3-1. 플레이어 캐릭터 디자인 개선
**위치**: drawPlayer() 함수
**문제**: 노란색 사각형 + 검은 눈만으로는 정체성 부족
**해결 방법**:
- 픽셀 아트 느낌의 간단한 캐릭터로 그리기:
  - 현재: 20x20 사각형
  - 개선: 머리(8x8) + 몸(8x8) + 팔(2x6 x2) + 다리(2x6 x2)를 여러 사각형으로 조합
  - 점프할 때 팔이 위로 올라가도록 애니메이션

#### 3-2. 배경 레이어 추가
**위치**: draw() 함수, 플랫폼 그리기 전
**문제**: 배경이 단색 그라데이션만으로 시각적 깊이 부족
**해결 방법**:
- 원거리(back), 중거리(mid), 근거리(front) 3개 레이어 그리기
- 플레이어 높이에 따라 레이어별 스크롤 속도 다르게 (시차 효과)
- 각 레이어에 간단한 도형 또는 grid 패턴 추가
- 스테이지별 배경 색상 테마 유지

#### 3-3. 플랫폼 시각적 다양화
**위치**: drawPlatforms() 함수
**문제**: 색상과 strokeStyle만으로는 타입 구분이 불충분
**해결 방법**:
- NORMAL: 줄무늬 무늬 (ctx.fillRect로 작은 선들)
- MOVING: 화살표 표시 또는 물결 모양
- SPRING: 코일 모양 (원 여러 개 그리기)
- DISAPPEARING: 체스판 패턴
- BREAKABLE: 균열 패턴 (지그재그 선)
- SLANTED: 각도 있는 사각형 + 화살표

### 현 상태 평가
Pixel Quest는 **견고한 기술 기반 위의 불완전한 기획 구현**입니다. Canvas 물리, 충돌 감지, 기본 게임 루프는 정상이지만, 설계서의 핵심 기능 9개 중:

- 미구현: 2개 (기능 5, 8, 9)

### 부족한 이유
1. **기획과 구현의 괴리**: 설계서는 야심찬 AI, 분석, 스코어보드를 언급했으나 구현 단계에서 제외됨

3. **시각적 피드백 부족**: 이펙트, 배경 레이어, 캐릭터 개성 미흡 → "AI slop" 수준은 아니지만 단조로움

### 개선 방향

- 독창성 4점(경계): 기획의 참신함은 있으나 구현 미흡
- 기능성 5점: 절반만 작동하는 게임
- 재검수 시 우선순위 1~3의 개선이 필수

**재검수 목표 점수**:
- 기능성 8/10 (핵심 기능 구현)
- 독창성 6/10 (시각적 다양화)

function generateAIAnalysis() {
    const deathsByStage = gameState.playerStats.deathsByStage;
    const stageClears = gameState.playerStats.stageClears;
    const totalDeaths = gameState.playerStats.deathCount;
    const playTime = (Date.now() - gameState.gameStartTime) / 1000;

    let analysis = [];
    
    // 분석 로직: 가장 많은 사망이 있던 스테이지 찾기
    let hardestStage = 1;
    let maxDeaths = 0;
    for (let i = 1; i <= 5; i++) {
        if (deathsByStage[i] > maxDeaths) {
            maxDeaths = deathsByStage[i];
            hardestStage = i;
        }
    }
    
    // 1. 사망 분석
    if (totalDeaths === 0) {
        analysis.push("완벽한 플레이! 실수 없이 모든 스테이지를 클리어했습니다!");
    } else if (totalDeaths <= 3) {
        analysis.push("매우 좋은 성적입니다! 조금의 연습으로 더 높은 점수를 기대할 수 있습니다.");
    } else {
        analysis.push(`스테이지 ${hardestStage}에서 ${maxDeaths}회 사망했습니다. 해당 스테이지의 플랫폼 배치를 다시 분석해보세요.`);
    }
    
    // 2. 콤보 분석
    if (gameState.maxCombo === 3) {
        analysis.push("3배 콤보를 달성했습니다! 뛰어난 회피 능력을 보여주셨습니다.");
    } else {
        analysis.push(`최대 콤보는 ${gameState.maxCombo}배입니다. 적을 피하는 데 더 집중하면 콤보를 높일 수 있습니다.`);
    }
    
    // 3. 속도 분석
    if (playTime < 120) {
        analysis.push(`${Math.round(playTime)}초 만에 클리어! 매우 빠른 플레이입니다.`);
    } else {
        analysis.push(`플레이 시간: ${Math.round(playTime / 60)}분 ${Math.round(playTime % 60)}초. 다음 시도에는 더 빠르게 진행해보세요.`);
    }
    
    return analysis.slice(0, 3);
}
