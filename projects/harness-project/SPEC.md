# 레트로 플랫포머: Pixel Quest

## 개요

Pixel Quest는 80년대 아케이드 감성의 2D 플랫포머 게임으로, 브라우저에서 바로 플레이할 수 있습니다. 플레이어는 다양한 플랫폼을 넘고, 적을 피하며, 보물을 수집하는 여정을 통해 순수한 게임플레이의 재미를 경험합니다. 점수, 콤보, 난이도 조절을 통해 진정한 도전이 무엇인지 보여주는 게임입니다.

## 디자인 방향

**분위기**: 1985년 아케이드 캐빈의 모니터에서 튀어나올 듯한 화려한 색상과 선명한 픽셀 아트. 네온 느낌보다는 따뜻한 튜브 모니터의 광채감.

**색감**: 
- 주요 팔레트: 밝은 노랑(캐릭터), 진한 파랑(배경), 형광 핑크/오렌지(플랫폼과 적)
- 배경은 조용한 남색 또는 검은색으로 유지하여 게임 요소가 돋보이게 함
- 각 스테이지마다 색상 테마 변경 (조명 변경의 느낌)

**톤앤매너**:
- 픽셀 폰트로 점수, 레벨, 생명 정보 표시
- 효과음과 함께 16비트 BGM으로 시간을 초월한 감각 자극
- 승리/실패 화면은 극적이고 직관적 (화면 깜빡임, 단순한 메시지)

**차별화 디자인**:
1. **동적 카메라**: 플레이어가 점프할 때 화면이 살짝 흔들려 충격감을 강조. 고도에 따라 배경 스크롤 속도 변경 (시차 효과).
2. **플랫폼 디자인의 다양성**: 고정 플랫폼, 이동 플랫폼, 사라지는 플랫폼, 스프링 플랫폼 등으로 시각적·기능적 다양성 제공. 각각 다른 색상 패턴(줄무늬, 물결, 반짝임)으로 역할 직관화.

## 기능 목록

### 기능 1: 플레이어 이동 및 점프 메커닉
- **설명**: 플레이어 캐릭터가 좌우 화살표 또는 WASD로 이동하고, 스페이스바나 위 화살표로 점프할 수 있습니다. 점프는 2단 점프(더블점프), 공중에서의 미세 방향 조정이 가능하며, 착지 시 충격파 이펙트로 피드백을 줍니다.
- **사용자 스토리**: 플레이어는 직관적인 컨트롤로 높은 정밀도의 플랫포밍을 경험하고, 각 점프 액션이 만족스러운 물리 반응을 제공하므로 숙련도에 따라 어려운 도전도 가능합니다.
- **포함 요소**: 
  - 좌우 이동 (가속/감속)
  - 기본 점프 + 2단 점프
  - 낙하 속도 제어 (버튼 홀드 시간에 따른 높이 조절)
  - 착지 충격 이펙트 (원형 파동, 먼지 입자)
  - 벽 타기 (선택사항): 수직 벽면에서 미끄러지며 점프 가능

### 기능 2: 플랫폼 시스템 (다중 타입)
- **설명**: 단순한 판자가 아닌 6가지 플랫폼 타입으로 레벨 디자인의 깊이를 제공합니다. 각 타입은 시각적으로 구별되며, 플레이어는 플랫폼의 외형으로 그 특성을 직관적으로 파악합니다.
- **사용자 스토리**: 플레이어는 플랫폼의 형태만 봐도 "이건 움직일 것 같은데?", "이건 사라질 거 같다"고 예상하며, 각 플랫폼의 특성을 활용해 수직으로 높이 올라가는 긴장감을 경험합니다.
- **포함 요소**:
  - 고정 플랫폼 (기본, 노란색 줄무늬)
  - 이동 플랫폼 (좌우 또는 상하 왕복, 파란색 물결 패턴)
  - 스프링 플랫폼 (착지 시 높이 점프, 형광 핑크)
  - 사라지는 플랫폼 (착지 2초 후 사라짐, 검은색 점프무늬)
  - 한 번만 사용 가능한 플랫폼 (부서지는 효과, 회색 균열)
  - 기울어진 플랫폼 (슬라이드 가능, 원목 갈색)

### 기능 3: 적 캐릭터 및 AI 행동
- **설명**: 3가지 적 타입이 서로 다른 행동 패턴으로 도전을 제시합니다. 각 적은 직관적 디자인으로 특성을 나타냅니다.
- **사용자 스토리**: 플레이어는 적의 행동 패턴을 학습하고, 그 사이사이를 재빠르게 통과하거나 점프로 넘어가는 순간의 쾌감을 느낍니다. 같은 레벨이라도 적의 배치에 따라 난이도가 크게 달라져, 재플레이 가치가 높습니다.
- **포함 요소**:
  - 순찰 적 (정해진 경로를 왕복, 빨간색, 느린 속도)
  - 플레이어 추격 적 (플레이어가 일정 거리 내 들어오면 추격, 주황색, 중간 속도)
  - 패턴 공격 적 (정해진 경로를 움직이며 주기적으로 발사, 보라색, 분사 이펙트)

### 기능 4: 점수 및 콤보 시스템
- **설명**: 단순히 적을 피하는 것이 아니라, 어떻게 피하느냐에 따라 점수가 달라집니다. 연속 도전(콤보)을 통해 플레이어에게 장기전 전략을 장려합니다.
- **사용자 스토리**: 플레이어는 "최대한 빠르게 레벨을 클리어하되, 모든 보물을 먹고, 적을 피하면 더 높은 점수"라는 멀티태스킹 목표로 중독성 있는 반복 플레이를 경험합니다.
- **포함 요소**:
  - 기본 점수: 레벨 클리어 (1000점)
  - 보물 수집 (동전 50점, 보석 200점)
  - 시간 보너스 (레벨 클리어 시간이 짧을수록 추가 점수)
  - 콤보 시스템 (연속 적 회피 시 배수 증가, 최대 3배)
  - 생명 보너스 (게임 끝날 때 남은 생명 * 500점)

### 기능 5: AI 기반 난이도 자동 조절
- **설명**: 플레이어의 성능을 학습하여 실시간으로 난이도를 조절합니다. 게임 중단 없이 플레이어가 느끼는 도전감을 최적화합니다.
- **사용자 스토리**: 초보자는 천천히 게임을 배우고, 숙련자는 계속 도전 받으며, 모든 플레이어가 자신의 수준에 맞는 긴장감 있는 경험을 합니다.
- **포함 요소**:
  - 성공률 추적 (클리어 시간, 피해 횟수, 획득 점수)
  - 동적 적 수 조절 (성공률이 높으면 다음 레벨에 적 추가)
  - 플랫폼 배치 변형 (난이도에 맞춰 위험한 플랫폼 비율 조절)
  - 플레이어 피드백 (현재 난이도 표시, "도전 상향 중" 같은 메시지)

### 기능 6: 스테이지 프로그레션 및 테마
- **설명**: 5개 이상의 스테이지가 각각 독립적인 시각적 테마, 플랫폼 조합, 적 배치를 가집니다. 플레이어는 각 스테이지의 변화를 통해 게임의 진행 느낌을 충분히 경험합니다.
- **사용자 스토리**: 플레이어는 "첫 번째 스테이지는 쉬운 튜토리얼, 마지막 스테이지는 모든 요소가 섞인 최종 도전"이라는 명확한 게임 호의 곡선을 느끼며, 각 스테이지 변화에 시각적 흥미를 유지합니다.
- **포함 요소**:
  - 스테이지 1: 기초 (고정 플랫폼만, 고정 적)
  - 스테이지 2: 이동 (움직이는 플랫폼 도입)
  - 스테이지 3: 위험 (사라지는 플랫폼, 추격 적 도입)
  - 스테이지 4: 혼돈 (모든 플랫폼 타입 혼합, 패턴 공격 적)
  - 스테이지 5: 극한 (고속 이동, 최소 대기 시간, 최대 적)
  - 각 스테이지별 컬러 테마 (스테이지 1: 초록/노랑, 스테이지 2: 파랑/진청, ..., 스테이지 5: 빨강/검은색)

### 기능 7: 생명 시스템 및 게임오버 메커닉
- **설명**: 플레이어는 3개의 생명으로 시작합니다. 적에 닿거나 화면 아래로 떨어지면 생명 1개 소비. 생명이 0이 되면 게임 오버. 일부 스테이지에서는 생명을 1개 회복하는 구조가 있습니다.
- **사용자 스토리**: 각 실수가 되돌릴 수 없는 후회로 남지만, 제한된 생명 안에서 레벨을 클리어할 수 있다는 희망이 플레이어를 계속 시도하게 합니다. 게임 오버 후에는 점수를 기준으로 순위표에 올라갈 기회가 있습니다.
- **포함 요소**:
  - 3개 시작 생명
  - 생명 회복 아이템 (별 모양, 드물게 출현)
  - 게임 오버 화면 (현재 점수, 도달 스테이지, 최고 점수 대비 순위)
  - 스테이지별 체크포인트 (다음 스테이지로 넘어갈 때 생명 1개 회복)

### 기능 8: 스코어보드 및 도전 기록
- **설명**: 로컬 스토리지에 플레이어의 게임 기록을 저장합니다. 하이스코어, 최고 도달 스테이지, 최고 콤보 등을 표시하여 플레이어가 자신의 성장을 추적합니다.
- **사용자 스토리**: 플레이어는 게임을 종료했다가 다시 시작해도 이전 기록이 남아있어, "어제보다 더 높은 점수를 노린다"는 명확한 목표를 갖고 게임을 재플레이합니다.
- **포함 요소**:
  - 로컬 하이스코어 (상위 10개)
  - 최고 콤보 기록
  - 최고 도달 스테이지
  - 총 플레이 시간
  - 플레이 통계 (시도 횟수, 평균 클리어 시간)

### 기능 9: AI 기반 플레이 분석 및 피드백
- **설명**: 게임 오버 후 AI가 플레이어의 게임플레이를 분석하여 개선 팁을 제공합니다. "자주 실수하는 플랫폼", "콤보를 잃는 순간", "점프 타이밍 개선 영역" 등의 인사이트입니다.
- **사용자 스토리**: 플레이어는 단순히 "실패했다"라는 정보가 아니라 "왜 실패했고, 다음에는 어떻게 개선할 수 있을까"라는 구체적인 방향을 제시받아, 게임을 더 깊이 있게 마스터하려는 동기가 생깁니다.
- **포함 요소**:
  - 사망 분석 (어느 구간에서 사망했는지, 그 구간의 난이도 평가)
  - 시간 낭비 구간 표시 (특정 플랫폼에서 오래 머물렀는지)
  - 콤보 손실 지점 (어디서 콤보가 끝났는지)
  - 개선 제안 (다음 시도 시 시도할 만한 전략)

## 기술적 고려사항

**성능**: 픽셀 아트 기반의 가벼운 2D 렌더링으로 60fps 목표. 적 AI는 가벼운 상태 머신 기반.

**반응성**: 터치 디바이스(모바일/태블릿) 지원으로 게임패드와 터치 입력을 동시 지원.

**저장**: 브라우저 로컬 스토리지를 활용해 오프라인 상태에서도 점수/기록 저장 가능.

**접근성**: 색상에 의존하지 않는 플랫폼 구분 (모양 다양화), 키보드 전체 조작 가능, 옵션 메뉴에서 난이도 수동 조절 가능.

## 성공 기준

1. 플레이어가 **처음 5분 내에** 게임의 목표와 조작을 직관적으로 이해합니다.
2. 첫 스테이지 클리어 후 **자발적으로 재시도**하는 의욕이 생깁니다.
3. 플랫폼과 적의 다양성으로 인해 **같은 레벨도 재플레이마다 새로운 경험**을 제공합니다.
4. AI 난이도 조절로 모든 스킬 레벨의 플레이어가 **도전감 있는 경험**을 합니다.
5. 스코어보드와 피드백 시스템으로 플레이어가 **자신의 성장을 명확히 인식**합니다.
